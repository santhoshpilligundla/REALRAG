4 Code Changes Causing Performance Issues

Direct comparison of C:\RM-space\rm-web (reskin) vs C:\codebase\rm-web (develop). Files analysed: angular.json, package.json, index.original.html, styles.scss.

4.1 — Production Build: CSS Not Minified + Source Maps On

Reskin — angular.json (production)
"configurations": {
  "production": {
    "optimization": {
      "scripts": true,
      "styles": {
        "minify": false,
        "inlineCritical": false
      }
    },
    "sourceMap": true,
    "buildOptimizer": true
  }
}
"defaultConfiguration": ""
Develop — angular.json (production)
"configurations": {
  "production": {
    "optimization": true,
    "sourceMap": false,
    "buildOptimizer": true,
    "aot": true
  }
}

The reskin production build ships CSS completely unminified (minify: false). This is why styles.css is 255 KB instead of ~80 KB. inlineCritical: false forces an extra network round-trip before first paint. defaultConfiguration: "" means a plain ng build runs with optimization: false — if the OCR deployment script does not explicitly pass --configuration=production, the entire app ships in unoptimized dev mode.

5 Performance Fix List — Reskin

All performance and page-loading fixes identified across runtime analysis and code review.

JavaScript & Bundle

J1 Split the 2.5 MB monolithic main.js into lazy-loaded chunks P0
Angular code-splitting must be applied so only the current route's code downloads on load. The current build puts all feature code into one file that takes 26 seconds to parse. Each feature module should use loadChildren lazy loading.
angular.json + feature routing modules
J2 Remove legacy JS libraries from index.original.html P0
jQuery, jQuery UI, jqGrid, RAUL v2.85.5 JS (×2), Bootstrap JS, Tether, Popper are all render-blocking synchronous scripts still loading on every page. These must be removed once the Angular reskin replaces their functionality. 4.4 MB of duplicate JS.
src/index.original.html
J3 Deduplicate getPropertyFeatures API call P1
The same endpoint fires twice per page load (3.5 sec + 3.2 sec = 6.7 sec wasted). Two components are independently subscribing to the same data. Use shareReplay(1) in the service or a shared state store to deduplicate.
getPropertyFeatures service
J4 CommonJS libraries (ng2-img-max, ng2-pica, raf) not tree-shakeable P1
These three libraries are listed in allowedCommonJsDependencies, bypassing Angular's tree-shaking warning. CommonJS modules are always fully bundled. Replace with ESM equivalents or ensure they are only imported in the specific lazy-loaded modules that need them.
angular.json — allowedCommonJsDependencies
J5 Load Google Maps API asynchronously P2
Google Maps is loaded synchronously without loading=async, blocking HTML parsing until the Maps script downloads and executes. Add loading=async to the Maps script tag.
src/index.original.html or Maps initialisation service

Build Configuration

B1 Enable CSS minification in production build P0
Change "styles": { "minify": false } to "styles": { "minify": true } in the production configuration. This alone will reduce styles.css from 255 KB to ~80 KB.
angular.json — configurations.production.optimization.styles
B2 Enable critical CSS inlining P0
Change "inlineCritical": false to true. Angular will extract above-the-fold CSS and inline it in the HTML, allowing the browser to render the first frame without waiting for the full stylesheet to download.
angular.json — configurations.production.optimization.styles
B3 Set a defaultConfiguration to prevent unoptimized deployments P1
"defaultConfiguration": "" means a plain ng build runs with optimization: false, buildOptimizer: false, and full source maps. If the OCR deployment script does not pass --configuration=production explicitly, the app ships in dev mode. Set "defaultConfiguration": "production".
angular.json — defaultConfiguration
B4 Disable source maps in production P1
"sourceMap": true in the production config generates .map files that are served alongside every JS chunk on first load. Set "sourceMap": false in the production configuration as done in the develop branch.
angular.json — configurations.production.sourceMap

CSS

C1 Move omnibar CSS from dev CDN to production CDN and reduce file size P0
omnibar.css is 1.3 MB and served from cs-cdn-dev.realpage.com — a dev origin with no edge caching. Takes 8,540 ms on first load. It must be published to cs-cdn.realpage.com (production CDN) and all dev/debug styles stripped. Target size: under 100 KB.
Omnibar package publishing configuration
C2 Fix Cache-Control headers on omnibar CSS P0
Missing cache headers cause the browser to re-download the 1.3 MB omnibar CSS on every SPA navigation (2,341 ms each time). In QA the second load is 414 ms. Set Cache-Control: public, max-age=31536000, immutable on the production CDN for versioned assets.
cs-cdn-dev.realpage.com server configuration
C3 Configure Tailwind content purging to remove unused utility classes P1
@tailwind utilities in styles.scss generates every possible utility class (2–3 MB unminified). The content array in tailwind.config.js must list all component file paths so Tailwind's JIT compiler removes unused classes. Combined with minify: true this will reduce the global CSS bundle significantly.
tailwind.config.js — content array
C4 Lazy-load theme token files — only one theme is active at a time P1
Three token files (tokens-legacy.css, tokens-light.css, tokens-dark.css) are all bundled into the global stylesheet and downloaded on every page load. Only one theme is active. Load only the active theme's tokens on startup and swap dynamically.
angular.json styles array + theme service
C5 Consolidate CSS to fewer domains — eliminate extra TCP/SSL handshakes P1
OCR loads CSS from 8 domains causing 5 extra TCP+SSL handshakes (~500–1,000 ms each). The reskin added cs-cdn.realpage.com alongside existing cdn.realpage.com and cs-cdn-dev.realpage.com. Consolidate to one CDN domain for all RealPage-hosted CSS.
index.original.html + omnibar config
C6 Enable HTTP/2 on the OCR app server P2
19 of 32 CSS files have 300–800 ms server wait times despite being under 15 KB. Files this small should respond in under 50 ms. HTTP/2 multiplexing allows all CSS requests over one connection to be served simultaneously without queuing.
OCR server infrastructure configuration
C7 Add <link rel="preload"> for critical CSS files P2
Critical stylesheets (omnibar.css, styles.css, raul.css) are discovered late during HTML parsing. Preload hints tell the browser to fetch them in parallel with the HTML document, reducing time-to-first-render by 1–2 seconds.
src/index.original.html — <head>

Layout Stability

L1 Reserve space for components before data loads — fix CLS score of 2.77 P0
Content is jumping 27× beyond the acceptable limit (threshold: 0.1, OCR score: 2.77) as reskin components load in. Dashboard widgets must use skeleton placeholders or fixed-height containers so layout does not reflow as data and components arrive. Primary contributors: omnibar, Angular dashboard widgets, GridStack reflow.
Dashboard component templates
L2 Fix ag-Grid sizeColumnsToFit() called on hidden containers (9 occurrences) P1
The reskin's new layout changes container visibility timing, causing grids to call sizeColumnsToFit() while their container has zero width. Each failed sizing causes a wasted render cycle. Call sizeColumnsToFit() only after the gridReady event and use a ResizeObserver to re-fire when the container becomes visible.
Grid component ngAfterViewInit hooks
