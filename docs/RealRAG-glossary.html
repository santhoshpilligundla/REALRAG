RealRAG — Glossary & Buzzwords

Hand this to the team. Each row is a term to learn + a one-line plain-English explanation + why it matters for RealRAG.
CategoryTermWhat it does / meansWhy it matters here
RAG patternsRAG (Retrieval-Augmented Generation)LLM answers a question using retrieved chunks of context, not just its training data.The core pattern of the whole system.
Naive RAGOne retrieval → one LLM call → one answer.Baseline path for simple factual questions (~70% of queries).
Self-RAGThe LLM critiques its own retrieval, asks for more if weak, then answers.Lifts accuracy on ambiguous queries.
Agentic RAGLLM equipped with tools (search, graph-walk, read-file) plans a multi-step investigation.Needed for multi-hop / cross-repo / "why" questions.
GraphRAG (Microsoft, 2024)RAG over a knowledge graph; cluster the graph, summarize each community.Best pattern for "explain the whole module / system" questions.
CRAG (Corrective RAG)If retrieval is insufficient, fall back to a live web/source walk.Our doc-first / code-fallback strategy.
HyDE (Hypothetical Document Embedding)Generate a fake answer first, embed that, search with it.+15-25% recall on vague or paraphrased questions.
RAG triadThree RAG accuracy dimensions: context relevance, groundedness, answer relevance.The metrics every RAG system must track.
Lost-in-the-middleLLMs attend more to start/end of context, less to the middle.Order retrieved chunks deliberately; don't dump 30 chunks.
Embeddings & vectorsEmbeddingA vector (e.g., 3072 floats) representing the meaning of a chunk of text.The "address" by which we find similar content.
Cosine similarityAngle-based distance between two vectors. Closer = more semantically similar.The score FAISS returns; threshold below 0.40 = noise.
FAISS (Facebook AI Similarity Search)A C++/Python library for fast nearest-neighbor search over millions of vectors.Our vector DB; per-repo IVF index.
IVF (Inverted File Index)FAISS index type that clusters vectors first, then searches the closest clusters.Fast; good default for < 10M vectors.
HNSW (Hierarchical Navigable Small World)Graph-based approximate-NN index; very fast, slightly more memory.Alternative to IVF when latency is critical.
Multi-vector embeddingEmbed signature, body, summary separately; score them independently.+10-20% recall when question type varies.
ColBERTLate-interaction multi-vector retriever — token-level matching at query time.State-of-the-art for high-precision retrieval; heavier infra.
text-embedding-3-largeOpenAI's high-quality embedding model (3072 dims).What we use for code + doc embeddings.
Embedding cacheHash chunk content; skip re-embed if unchanged.Cuts re-index cost by >99% on incremental updates.
Retrieval techniquesBM25 / FTS (Full-Text Search)Classic keyword scoring; Postgres has it built-in via tsvector.Catches exact phrases that vector search misses.
Hybrid retrievalRun vector + keyword + symbol-exact in parallel; merge results.Single biggest accuracy lever — covers all question types.
Re-rankingTake top-50 from retrieval, re-score with a smarter (slower) model, keep top-5.+15% accuracy at small latency cost.
Cross-encoderModel that scores a (query, chunk) pair jointly. Slow per pair but very accurate.The standard re-ranker (e.g. BAAI/bge-reranker-v2-m3).
MMR (Maximal Marginal Relevance)Diversity filter — don't return 5 near-duplicate chunks.Covers more ground; better answers on broad questions.
Score floorDrop any chunk below a similarity threshold (e.g. 0.40).Stops noise from polluting the LLM prompt.
LLM patternsPrompt engineeringCrafting the system + user message so the LLM behaves as wanted.Single highest-leverage skill in this whole project.
Few-shot promptingInclude 2-3 examples of the desired output in the prompt.Locks in voice and structure across the doc corpus.
Chain-of-thought (CoT)Ask the LLM to reason step by step before answering.Lifts accuracy on complex code / multi-hop questions.
ReAct (Reason + Act)The agentic loop: think → call a tool → observe → think → answer.How agentic RAG actually works.
Tool use / function callingLLM emits structured JSON to call a real function (search, read-file).Mechanism behind agentic source-walking.
Structured outputForce LLM to fill a JSON schema (Pydantic / Instructor / Anthropic tool use).Guarantees doc-gen completeness — no missing fields.
Prompt cachingAnthropic caches the system prompt + retrieved context for ~5 min.~60% token savings on follow-up turns.
Tier routingPick Haiku / Sonnet / Opus based on question complexity.30-100× cost savings vs. always-Opus.
Refusal modeSystem prompt forces the LLM to say "I don't know" if evidence is thin.Wrong answers become visible (refusals), not silent (hallucinations).
Doc generationParser-first generationExtract everything an AST/parser can give before invoking the LLM.Eliminates a whole class of hallucinations.
Schema-constrained generationForce LLM to fill a typed schema (every field required).Single best practice for "nothing missed" docs.
Diátaxis frameworkFour doc types: Reference, Explanation, How-to, Tutorial. Each answers different questions.Generate all four per entity — covers any question.
Hierarchical summarizationfunction → class → module → repo → system, each layer summarizing the one below.How "explain the whole repo" answers are produced.
Map-reduce summarizationChunk a big file, summarize each chunk, then summarize the summaries.Defeats lost-in-the-middle on huge files.
Chain-of-densityIterative: write brief → expand → expand → fill gaps.Salesforce/Anthropic technique for rich, dense docs.
Self-consistencyGenerate the same doc N times with high temperature; take consensus.Cuts factual variance.
Self-critique passSecond LLM call: "find errors against the source" → revise.Reduces factual errors ~40%.
Verifier passCheap deterministic check: each claim must be grounded in a real chunk.Drops hallucinated sentences before they reach the user.
Citation groundingEvery claim cites file:line; refusal if no source exists.Wrong answers become visibly wrong.
Knowledge-graph extractionPull (entity, predicate, value) triples from code/docs into a structured table.Deterministic answers for "what writes to X?" type questions.
Code-specifictree-sitterMulti-language parser library that produces a concrete syntax tree.The de-facto standard for code chunking.
AST (Abstract Syntax Tree)Parsed representation of source code — what tree-sitter produces.Lets us chunk at function/class boundaries, not arbitrary lines.
AST-aware chunkingSplit code at logical boundaries (function, class) instead of line counts.Preserves logic; embeddings make sense.
SCIP / LSIFSourcegraph's standard formats for cross-repo symbol graphs.Industry-standard "where is X used" data model.
LSP (Language Server Protocol)Microsoft's protocol for IDE-grade symbol info (definitions, references, types).Backbone of every modern IDE; great for fact extraction.
Symbol graphNodes = entities (functions, classes); edges = calls/imports/extends.The structural backbone for graph-walks and cross-repo expansion.
Dependency graphStored as a Postgres table of (from_entity, to_entity, kind).Enables agentic graph traversal at query time.
Indexing & updatesIncremental indexingRe-process only files that changed since last indexed SHA.Makes commit-driven freshness affordable.
Entity-content hashHash function/class body; skip re-embed/re-doc if unchanged.A whitespace edit doesn't trigger LLM cost.
Atomic upsertPostgres + FAISS updated in a single transaction; rollback on failure.Prevents half-indexed corrupted state.
Idempotent webhookDedupe on (repo_id, commit_sha) — replays are safe.Webhook retries can't double-index.
Dead-letter queue (DLQ)Failed indexing jobs land here for manual review / retry.Errors don't silently disappear.
Index versioningTag every chunk with (embedding_model, schema_v, parser_v).When any of these changes, force rebuild.
EvaluationGolden test set100-200 hand-curated Q→A pairs to regression-test the system.Catches accuracy regressions when prompts/retrieval change.
RAGASOpen-source RAG eval framework — metrics for faithfulness, relevance, etc.Standard tool to measure RAG quality.
Faithfulness"Does the answer's claims appear in the cited chunks?"The anti-hallucination metric.
Context precisionOf the retrieved chunks, how many are actually relevant?Measures retrieval signal-to-noise.
Context recallOf all the relevant chunks in the corpus, how many did we retrieve?Measures retrieval coverage.
Answer relevance"Did the answer actually address the question?"End-to-end user-facing quality metric.
Architecture & opsStreaming / SSE (Server-Sent Events)LLM tokens streamed to the UI as they're generated.First word in <1s even if total answer takes 5s.
Async / asyncioPython concurrency model that lets us run multiple retrievers in parallel.Keeps total retrieval latency = max(retrievers), not sum.
Connection poolReuse Postgres connections instead of opening a new one per request.Cuts per-query latency by 50-200ms.
HMAC verificationCryptographic check that a webhook payload came from TFS.Stops anyone from triggering re-indexing maliciously.
OpenTelemetry (OTEL)Vendor-neutral standard for traces / metrics / logs.Single way to instrument every layer for observability.
Blue/green deployRun two versions side-by-side; switch traffic when new is healthy.Zero-downtime rollouts; instant rollback.
Frameworks & toolsFastAPIModern Python web framework with built-in OpenAPI + async.Our backend.
Pydantic v2Type-safe data models / validation in Python.Config + DTOs + structured-output enforcement.
psycopg v3Modern Postgres driver for Python (async + sync).Our DB driver.
LangChain / LlamaIndexPython frameworks for building RAG / agent pipelines.We don't use them directly — but they're reference impls for every pattern.
DSPy (Stanford)Programmable prompt pipelines; treats prompts like code modules.Powerful for evolving the doc-gen pipeline scientifically.
InstructorLibrary for forcing LLMs to return Pydantic models.Implements schema-constrained output cleanly.
Sourcegraph Cody / Cursor / ContinueProduction code-RAG products to study.Reference implementations of context-aware doc-gen.
Microsoft GraphRAGOpen-source impl of the GraphRAG paper.Canonical reference for the graph-RAG pattern.
MintlifyAI dev-doc generator (commercial).Reference for how doc-gen ships in industry.
StreamlitPython framework for fast data/AI UIs.Our demo chat UI.
Quality & safetyHallucinationLLM produces content not grounded in any real source.The #1 risk in any RAG system.
GroundednessEvery claim in the answer is supported by a retrieved chunk.The thing citation enforcement guarantees.
Sensitive-data filterScan and drop secrets / PII / credentials before indexing.Stops keys/passwords from being embedded into FAISS.
trufflehog / detect-secretsOpen-source tools to find secrets in code.Run before indexing, every commit.
Audit logPersistent record of (user, question, answer, citations, time).Compliance, abuse detection, improvement input.
Domain glossary injectionPre-pend known business terms (propcode, RRG) into every prompt.Forces consistent vocabulary; prevents synonym drift.
Doc style frameworksDiátaxis (Daniele Procida)Four-doc-type framework: Reference / Explanation / How-to / Tutorial.Adopted by Django, GitLab, Cloudflare, NumPy.
Google / Microsoft writing style guidesIndustry-standard prose rules for technical docs.Inject relevant rules into doc-gen prompts.
JavaDoc / JSDoc / docstringsInline structured comments; carry author intent.Always include verbatim — never paraphrase.
How to use this: share with the team. Each person picks 5-10 unfamiliar terms a day, reads the linked papers/docs, then we sync. By the end of a week, everyone speaks the same language and we can architect together without confusion.
