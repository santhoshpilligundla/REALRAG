📁 RealRAG File Purpose Guide

What every file does and why it exists

🤖 Chat & Query
lib/chat.py
Main chat orchestrator. Implements the 4-tier auto-escalation router (Fast → Deep → Agent → Live). Decides which mode to use, validates answers for faithfulness, and streams responses to the UI.
Core
lib/retrieval.py
Hybrid search engine. Runs 5 methods in parallel (vector, keyword, symbol, knowledge-graph, doc-vector), deduplicates results, and returns the top ranked hits with citations.
Core
lib/agent.py
Tier-3 agentic mode. Claude plans and executes multi-step investigations using tool-use: search, symbol lookup, KG query, code grep. Best for complex cross-repo questions.
Agent
lib/agent_flows.py
Flow router for Tier-4 Live mode. Reads product dependency graphs from agent_flows.yaml, classifies which repo chain to investigate, and builds the execution plan.
Agent
lib/claude_runner.py
Headless Claude Code agent runner. Launches existing Claude Code agents (from .claude/agents/ folders in repos) without any UI, streaming their output back to the chat.
Agent
lib/query_cache.py
Exact question-to-answer cache. Stores confirmed answers keyed by normalized question + product ID. Returns instant results for repeated questions, bypassing all LLM calls.
Performance
lib/sessions_repo.py
Chat session persistence. Saves conversations to PostgreSQL so chat history survives page reloads and server restarts. Supports multiple sessions per user.
Persistence
lib/intent.py
Question intent classifier. Determines whether a question is about code structure, business logic, data flow, or configuration — used to tune retrieval strategy.
Routing
⚙️ Indexing Pipeline
lib/chunker.py
Walk-parse-chunk orchestrator. Traverses repo directories, detects file types, dispatches to the correct parser, and stores normalized ParsedEntity records in PostgreSQL.
Pipeline Step 2
lib/doc_gen.py
Pass-1 doc generator. Uses Claude to write 6-perspective documentation (structural, behavioral, business, edge cases, examples, cross-refs) for each parsed code entity.
Pipeline Step 3
lib/doc_gen_pass2.py
Pass-2 module rollup. Aggregates Pass-1 entity docs into per-file module summaries using Claude Sonnet, providing a higher-level view of each source file.
Pipeline Step 3b
lib/doc_gen_pass3.py
Pass-3 cross-module narratives. Uses Claude Opus to write cross-repo narrative docs based on cross-repo edges. Most expensive step — run after edges are built.
Pipeline Step 3c
lib/doc_gen_l4_enrich.py
L4 enrichment. Adds worked examples, statement annotations, and git history context to existing entity docs — deepens understanding for complex entities.
Pipeline Step 3d
lib/embedder.py
OpenAI embedding service. Sends text batches to text-embedding-3-large, caches vectors on disk (keyed by SHA1), handles 8000-token truncation and retry logic.
Pipeline Step 4
lib/embed_pipeline.py
Embedding pipeline coordinator. Fetches all pending chunks from DB, calls embedder in batches, and writes vectors to FAISS indexes. Tracks completion per repo.
Pipeline Step 4
lib/faiss_store.py
FAISS vector index management. Builds, reads, and searches per-repo FAISS indexes stored at storage/faiss/. Maintains chunk-id mappings for citation lookups.
Pipeline Step 5
lib/clone.py
TFS repo cloner. Clones repositories from TFS using PAT authentication, supports shallow cloning for speed, and tracks clone SHA for change detection.
Pipeline Step 1
lib/cross_repo.py
Cross-repo edge discovery. Runs pattern catalog across all repos to find dependencies (service calls, shared DB tables, common interfaces) and stores them as typed edges.
Pipeline Extra
lib/kg.py
Knowledge graph query engine. Answers deterministic structural questions (what writes to table X? what calls method Y?) via SQL triple queries — no LLM needed.
Search
🔬 Language Parsers
lib/parser_java.py
Parses Java source files using tree-sitter. Extracts classes, methods, interfaces, annotations, imports, and Javadoc. Handles Spring annotations for service detection.
Java
lib/parser_typescript.py
Parses TypeScript/Angular files using tree-sitter. Extracts components, services, interfaces, decorators. Understands Angular-specific patterns.
TypeScript
lib/parser_sql.py
Parses SQL stored procedures, views, and functions using sqlglot. Extracts business logic, table dependencies, and parameter documentation.
SQL
lib/parser_ddl.py
Parses database schema DDL files. Extracts table definitions, column types, constraints, and foreign keys — builds the data model for KG queries.
DDL
lib/parser_xml.py
Parses Spring XML configs and application configs using lxml. Extracts bean definitions, service wiring, property values, and component dependencies.
XML
lib/parser_jrxml.py
Parses JasperReports template files (.jrxml). Extracts report parameters, fields, data sources, and layout — documents report structure for chat queries.
Reports
lib/parser_markdown.py
Parses Markdown documentation files. Extracts sections, headings, and content blocks to make existing documentation searchable via RAG.
Docs
lib/parsers_common.py
Shared parser utilities. Provides common entity types, text cleaning, line-number helpers, and the ParsedEntity dataclass used by all language parsers.
Shared
🏗️ Infrastructure
lib/db.py
Database connection manager. Starts the embedded pgserver, creates the realrag database if missing, and provides the psycopg connection factory used everywhere.
Database
lib/config.py
Settings loader using pydantic-settings. Reads .env file for API keys, model names, DB paths. Provides a cached settings singleton accessible project-wide.
Config
lib/llm.py
LLM abstraction layer. Wraps Anthropic Claude API calls with retry, streaming, and JSON-mode support. Supports swapping models (Haiku/Sonnet/Opus) per call type.
LLM
lib/models.py
Pydantic data models. Defines RepoOnboardingRequest, ParsedEntity, and other shared data structures used across the pipeline and API boundaries.
Models
lib/repos_repo.py
Repository metadata CRUD. Handles inserting, updating, listing, and deleting repo records in PostgreSQL. Tracks clone status, parse counts, and indexing progress.
Database
lib/runs_repo.py
Pipeline run tracker. Records start/stop/error events for each pipeline stage (clone, parse, embed) so the Activity tab can show what ran, when, and how long it took.
Observability
lib/walker.py
File system walker. Traverses repo directories respecting .gitignore rules, file-size limits, and extension filters to produce the file list for chunker.py.
Pipeline
lib/coverage.py
Coverage metrics calculator. Computes per-repo indexing stats (Pass-1 coverage %, verified %, module docs, etc.) for the Coverage tab dashboard.
Metrics
🖥️ UI & Entry Points
frontend/streamlit_app.py
The entire Streamlit application. Contains all 5 tabs: Chat, Add Repos, Registered Repos, Coverage, Activity. WhatsApp-style chat bubbles with streaming responses.
UI
run_realrag.bat
One-click app launcher. Clears the stale PostgreSQL PID file if present, then launches streamlit run on the correct file. Run this every time to start the app.
Launcher
setup.bat
First-time setup script. Creates .venv, installs all dependencies from requirements.txt, and copies .env.example to .env. Run once after cloning.
Setup
scripts/dev_up.py
Developer bootstrap. Applies DB schema, seeds product data, and verifies the stack is healthy. Run after setup.bat if you need to reset the database.
Setup
📊 Data & Config
data/glossary_rms.yaml
RMS business term glossary. Defines canonical meanings for terms like "YieldStar", "renewal", "effective rent", "LRO". Injected into LLM prompts to ensure consistent terminology in answers.
Business Logic
data/agent_flows.yaml
Product dependency graphs for Live Agent mode. Defines which repos chain together per product (e.g., rm-web → ys → po). Adding new products requires only YAML — no code changes.
Routing Config
lib/presets.py
RMS repo preset metadata. Pre-filled display names, descriptions, workflows, and concepts for the 5 core RMS repos. Enables bulk-register in one click from the UI.
Config
lib/glossary.py
Glossary bootstrapper. Reads glossary_rms.yaml and inserts/updates terms in PostgreSQL on startup. Glossary terms are included in retrieval context for business questions.
Business Logic
lib/business_docs.py
Business HTML doc indexer. Embeds the RealRAG bible, scope, and glossary HTML files into a standalone FAISS index. Provides a "business language" search arm separate from code.
Business Logic
.env.example
Environment variable template. Lists all required keys: ANTHROPIC_API_KEY, OPENAI_API_KEY, model names, DB paths. Copy to .env and fill in before running.
Config
